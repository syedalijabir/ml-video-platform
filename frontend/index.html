AWS Architecture
How the ML video search platform runs on AWS

Infrastructure Components:
Application Load Balancer (ALB): Distributes incoming traffic across ECS tasks, provides SSL termination and health checks
ECS - API Service: Containerized FastAPI application with auto-scaling based on CPU/memory metrics. Handles video upload, search queries, and playback endpoints
CLIP Image Model (Hugging Face): OpenAI's CLIP model converts video frames and search queries into embeddings for semantic matching
Pinecone Vector Database: Stores 512-dimensional embeddings generated by CLIP model. Enables semantic similarity search across video frames
Amazon RDS (PostgreSQL): Relational database storing video metadata, processing job status, and frame timestamps
ECS - Worker Service: Background processing tasks that scale automatically based on SQS queue depth. Extracts frames from videos and generates embeddings using CLIP
Amazon SQS: Message queue for asynchronous video processing jobs. Triggers worker auto-scaling when queue depth increases
Amazon ECR: Private container registry storing Docker images for both API and Worker services
Amazon S3: Object storage for uploaded videos and extracted frame images. Serves videos via pre-signed URLs
